Privacy & Security

Understanding how your data is protected

Data Protection Overview

Your inventory data is stored and managed on Southern Cross University (SCU) servers. This page explains how we protect your information and maintain privacy.
Data Storage

Server Location

All inventory data is stored on SCU (Southern Cross University) servers located in Australia.

Data Persistence

Data persists across sessions and is available whenever you access the application from any device.

Auto-Generated IDs

Item IDs are automatically generated by the server to ensure uniqueness and prevent conflicts.
Data Transmission

HTTPS Encryption

All data transmitted between your device and the server is encrypted using HTTPS (SSL/TLS).

API Security

RESTful API endpoints require proper request format and use standard web security protocols.

Input Sanitization

User inputs are validated and sanitized on both client and server sides to prevent injection attacks.
User Privacy

Data Isolation

Each user has their own data view. You can only access and modify your own inventory records.

No Personal Tracking

We do not collect personal information beyond what is necessary for the inventory management functionality.

Delete Protection

Certain system items (like Laptop) cannot be deleted to maintain data integrity for all users.

 Security Best Practices
Use unique, descriptive item names
Keep inventory quantities accurate
Use Special Notes for important details
Mark featured items appropriately
Report any security concerns to SCU IT
